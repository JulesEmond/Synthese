Mini-Tutoriel Angular 10

Qu'est-ce qu'Angular 10?

Angular 10 est en fait la neuvième version de la deuxième plateforme Angular. Pourquoi deux plateforme? Parce que la première fonctionne grâce à Javascript et la seconde Typescript. Pour faire simple, Angular est un framework qui aide à la création de Single Page Application (SPA).

Exemple de commandes :

  • ng new nom-projet : Sert à créer un nouveau projet Angular dans le dossier nom-projet.
  • ng g c nom-composant : Sert à créer un nouveau composant nommé nom-composant.
  • ng g s nom-service : Sert à créer un nouveau service nommé nom-service.
  • ng g module nom-module : Sert à créer un nouveau module nommé nom-module.
  • ng serve -o : Sert à démarrer et lancer le projet Angular sur le Web (en localhost).

Mini-Tutoriel sur le Crud

Qu'est-ce qu'un CRUD?

CRUD est un accronyme pour Create-Read-Update-Delete qui sont les quatres fonctions de base qui doivent être implémenté dans toute base de donnés.

Fonction de base du CRUD :

  • CREATE : Ajoute un objet dans la base de données.
  • READ : Lit une partie des objets de la base de données qui peut être autant un seul objet ou l'entièreté des objets se trouvant dans cette dernière.
  • UPDATE : Modifie les données contenues dans un ou plusieurs objets de la base de données.
  • DELETE : Supprime un ou plusieurs objets de la base de données.

Mini-Tutoriel sur les Relations

Qu'est-ce qu'une relation?

Une relation est le lien qui lie deux objets ensemble dans une base de données.

Les types de relations :

  • Héritage : Désigne une relation parent-enfant.
  • Implémentation : Un des objets se sert de l'autre objet qui une interface ou un enum.
  • Association : Un objet se sert de l'autre.
  • Composition : Un objet fait nécessairement parti d'un autre.
  • Aggrégation : Un objet peut faire parti d'un autre.

Les types d'associations :

  • One To One : Désigne une association où un objet est lié à un autre.
  • One To Many : Désigne une association où un objet est lié à plusieurs objets d'un autre type.
  • Many To Many : Désigne une association où plusieurs objets d'un type sont liés à plusieurs objets d'un autre type.
